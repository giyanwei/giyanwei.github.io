Python Make Life Easier - Python - Flask框架

Created2024-07-11|Updated2024-07-11
|Post Views:

Flask框架

  • 特点
    • 轻量级
    • 灵活性
    • 易拓展
    • Jinja2模板引擎，werkzurg协议
  • 优劣
    • 性能优于Django
    • 第三方库多，可拓展性强
    • 与关系型数据库更契合，尤其NoSQL
    • 更Pythonic，符合python哲学
  • 路由
    • 在django中，路由是浏览器访问服务器时，先访问的项目中的url，再由项目中的url找到应用中url，这些url是放在一个列表里，遵从从前往后匹配的规则。
    • 在flask中，路由是通过装饰器给每个视图函数提供的，而且根据请求方式的不同可以一个url用于不同的作用。
  • 视图函数
  • 蓝图
    • Flask组件化，把实现不同功能的module分开，把大的App分割成各自实现不同功能的module
    • 不能在应用创建后撤销注册一个蓝图而不销毁整个应用对象。
  • 上下文管理
    • 请求上下文
      • request
      • session
    • 应用上下文
      • current_app
      • g
    • flask上下文管理可以分为三个阶段：
      • 1、’请求进来时’：将请求相关的数据放入上下问管理中
      • 2、’在视图函数中’：要去上下文管理中取值
      • 3、’请求响应’：要将上下文管理中的数据清除
  • 请求钩子
    • 装饰器实现
      • 1，before_first_request 在处理第一个请求前运行
      • 2，before_request:在每次请求前运行
      • 3，after_request:如果没有未处理的异常抛出，在每次请求后运行
      • 4，teardown_request:即使有未处理的异常抛出，在每次请求后运行
  • 用过的Flask库
    • flask_SQLAlchemy
    • flask_login
    • flask_cors
